SquareGPS Test Assignment

You need to build an application using Vuejs, Vuex, VueRouter, Vuetify for this test assignment.

The application's header should contain a menu with 2 items: About the Task and Map.

The main part of the application should display the content of these sections.

Section "About the Task"

In this section, there should be the text of this assignment, formatted similarly.

Section "Map"

The screen should display a map and a list (see example):

1. Clicking the add button should switch the map to marker addition mode: a marker should appear at the click location, and in the list - a record with the point's address.

For address search, you can use any free geocoding API, for example: https://geocode.maps.co/

In case the address is not found, an error should be displayed, and the marker should not be added to the list.

2. Clicking on a marker should highlight the corresponding entry in the list.

3. Clicking on an entry in the list should center the map on the marker.

4. Markers should be stored locally and should not disappear after page reload.

5. The selected marker's ID should be displayed in the address bar.

Local data storage should be organized in the form of a service that emulates the backend. (A class Backend is required, which pseudo-asynchronously saves data to localStorage)

Additional:

1. Consider mobile device display.

2. Provide localization.

3. You need to write 1 unit test for any component using jest.

Reminder: This test assignment is intended to demonstrate the ability to write good modular code and to use the specified technologies and more. Be proud of this code.
